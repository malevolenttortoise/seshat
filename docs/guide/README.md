# Seshat — operator and power-user guide

The chapters under `docs/guide/` cover everything an operator or power-user needs once Seshat is running: the multi-author/series model, per-source metadata behavior, the two metadata caches, the Metadata Manager review surface, hygiene jobs, notification routing, and active replacement. For installation and first-boot configuration start at the root [`README.md`](../../README.md) and [`DEPLOY.md`](../../DEPLOY.md); this guide picks up from there.

## Chapters

1. [Multi-author and series](./multi-author-and-series.md) — contributor sets, primary author at position 0, series `author_mode` from contributor intersections, owner-vs-incidental, heal-on-convergence, the Persons & IDs page, and source-ID-aware author identity.
2. [Metadata sources](./metadata-sources.md) — per-source behavior for the nine discovery sources (Goodreads, Hardcover, Amazon, Audible, Open Library, Google Books, Kobo, IBDB, MAM): what each covers, priority order, failure modes, and tuning knobs. Includes the Goodreads Cloudflare bypass and the resolver chain.
3. [Metadata cache](./metadata-cache.md) — the two-cache architecture (Amazon detail cache + Goodreads list-page cache), their divergent worker postures, first-fill UX, status surfaces, operator interventions, and settings keys.
4. [Metadata Manager](./metadata-manager.md) — the owned-book proposed-change review queue: where rows come from, the field-type filter chip row, the authors proposed-change flow, approve → push-back → inline re-sync.
5. [Hygiene jobs](./hygiene-jobs.md) — catalogue of the 12 hygiene jobs with what each does, when it runs, side effects, and operator pre-flight notes (Job 9 backup recipe, Job 11 first-run HEAD-verify cost).
6. [Notifications](./notifications.md) — the 21-event registry, hierarchical-dotted taxonomy, `prefix.*` + `*` wildcard routing, per-event priority, quiet hours, the Settings UI walkthrough, and the migration path from the pre-v2.28.0 `notify_on_*` keys.
7. [Active replacement](./active-replacement.md) — the opt-in upgrade flow: quality scoring inputs, safety gates, the soft-delete-to-`.seshat-replaced/<ts>/` reversibility model, sink path divergence, and the retention sweeper.

## Common issues

A short troubleshooting playbook for the issues that come up most often but don't justify their own chapter. Each entry is symptom → likely cause → fix or chapter pointer.

| Symptom | Likely cause | Fix |
| --- | --- | --- |
| Goodreads cache panel shows worker enabled, but nothing is scanning. | Worker is disabled by default and was just toggled on; the first author hasn't been popped from the queue yet, **or** the Goodreads dispatcher is in a budget-exhaust skip. | Wait one tick (≤30s) for the first scan to start. If the navbar stays grey after several minutes, check the `today_budget_exhaust_count` field — see [`metadata-cache.md` → Goodreads worker posture](./metadata-cache.md#goodreads-worker-posture). |
| Amazon cache stuck on cooldown for hours with the status pill yellow. | An `_abck` posture escalation. The Akamai cooldown is intentional; resetting it immediately re-triggers escalation. | Let it ride. If you're certain the network conditions have changed (e.g. you rotated VPN exit), the **Reset Cooldown** button under Settings → Sources → Amazon clears the state. See [`metadata-cache.md` → Amazon worker posture](./metadata-cache.md#amazon-worker-posture). |
| A series shows as **Shared** instead of **Per-author** after upgrading to v3.0.0. | Pre-3.0 thin rows in the series carry only a single contributor each, so the cross-book intersection drops to empty. | The next discovery scan that touches one of those rows heals it (additive union) and the series flips. See [`multi-author-and-series.md` → Heal-on-convergence](./multi-author-and-series.md#heal-on-convergence--pre-30-thin-rows-self-correct). |
| Persons & IDs page shows the same author as two rows (e.g. "Robert Heinlein" and "Robert A. Heinlein"). | Pre-v3.2 name-only matching split one person across two rows even though the underlying author rows share a Goodreads/Amazon ID. | Run Hygiene Job 9 (Consolidate persons by shared source ID). **Back the metadata DB up first** — person merges are hard to reverse. See [`hygiene-jobs.md` → Job 9](./hygiene-jobs.md#job-9--consolidate-persons-by-shared-source-id). |
| An expected notification never fires. | Either no routing rule matched (default rule is implicit only when no explicit rule covers the event), the master Notifications toggle is off, or quiet hours are silently suppressing it. | Walk the rule chain in Settings → Notifications top-to-bottom; explicit rules win over the implicit default. See [`notifications.md` → Resolution order](./notifications.md#resolution-order) and [`notifications.md` → Quiet hours](./notifications.md#quiet-hours). |
| An approved change in Metadata Manager didn't push back to Calibre/CWA. | The sink-specific push path failed silently for that field type, or the sink itself is unreachable. | Check the row's status badge. CWA and calibredb diverge for the authors field; see [`metadata-manager.md` → Push-back routing](./metadata-manager.md#push-back-routing) and [`metadata-manager.md` → Failure modes](./metadata-manager.md#failure-modes). |
| Active replacement detected an opportunity but it sits on the badge counter indefinitely. | The downstream sink is unreachable or the library on-disk root failed the safety classification. | Check container logs for the safety-classification line and the sink's last reachability check. See [`active-replacement.md` → Safety gates](./active-replacement.md#safety-gates) and [`active-replacement.md` → Sink path divergence](./active-replacement.md#sink-path-divergence). |
| qBit won't connect — URL/user/password are correct. | On qBit 5.1+, "Enable Host header validation" defaults ON and rejects Seshat's requests with 403. On qBit 5.2+ with an IP-whitelist that catches Seshat's source IP, the login can return HTTP 204 (which older Seshat builds didn't accept). | Disable Host header validation in qBit → Options → Web UI → Security, or set Server domains to `*`. If you see `qBit login failed: HTTP 204 body=''` in `docker logs seshat`, update to the latest image — the 204 response is handled as success. |
| Audiobookshelf library doesn't pull into Seshat regardless of folder layout. | Folder layout is irrelevant — Seshat reads ABS's API, not the filesystem. The library is either typed as Podcast (Seshat filters `mediaType=book` only), or the API token isn't saved correctly. | Confirm the ABS library type is **Book** in ABS → Settings → Libraries. Click the **Test Connection** button under Seshat → Settings → Audiobookshelf — it returns a real error message if the token is wrong. |
| Multiple editions of the same book all show as "missing" in Discovery. | Discovery enumerates editions from the metadata source's edition list (Goodreads/Hardcover/Amazon); the source's idea of "every edition" includes ones you already own. The review queue is non-destructive — entries sit there until you approve or dismiss. | This is review-only; nothing is grabbed without your approval. Cross-library Persons + format-priority dedup continue to tighten this surface; the queue is the place to hide edition variants you don't want to track. |
| Dashboard says "Dispatcher: Offline" right after startup. | A startup error stopped the orchestrator. The most common cause is a malformed setting in `settings.json` (often hand-edited). | Check `docker logs seshat` for the stack trace and restart. The container is safe to restart; data lives entirely on `/app/data`. |
